# Org Lens — Chat Module

A chat interface that lets users explore people, expertise, projects, and org structure through natural language queries. Powered by Google Gemini with function calling.

## How It Works

```
User asks a question
       │
       ▼
  Gemini LLM (with function calling)
       │
       ├── Decides which tool(s) to call
       │   (find_experts, get_person, search_people, etc.)
       │
       ▼
  DatasetIndex (in-memory)
       │
       ├── Queries pre-computed people/project data
       │
       ▼
  Gemini synthesizes tool results into a natural language answer
       │
       ▼
  SSE stream → Chat UI renders progressively
```

Gemini receives tool definitions describing what queries are available. When the user asks a question, Gemini decides which tools to call, the backend executes them against the in-memory DatasetIndex, and Gemini synthesizes the results into a conversational answer. The response streams back via Server-Sent Events for progressive rendering.

## Data Source

Org Lens does **not** fetch data from external APIs. It consumes pre-computed JSON files generated by the [site-analysis pipeline](https://github.com/red-hat-data-services/site-analysis):

```
site-analysis pipeline
  ├── people_summaries_{scope}.json   ──┐
  ├── people_categories_{scope}.json  ──┼──→  data/org-lens/{scope_id}/
  └── projects_{scope}.json           ──┘
```

The pipeline analyzes LDAP, Jira, and GitHub/GitLab activity, then uses Gemini to generate per-person summaries, expertise descriptions, and work categories. Org Lens loads these files into memory at startup and makes them queryable through the chat.

**Current data flow:** Files are manually copied from pipeline output into `data/org-lens/{scope_id}/`.

**Planned:** Automated upload via pipeline CronJob (see [pipeline upload design spec](https://github.com/red-hat-data-services/site-analysis/blob/main/docs/superpowers/specs/2026-06-18-pipeline-to-dashboard-upload-design.md)).

## Available Tools

The LLM can call these tools to answer user queries:

| Tool | Description | Example Query |
|------|-------------|---------------|
| `find_experts` | Find people with expertise in a topic | "Who knows Kubernetes?" |
| `get_person` | Get full profile for a person | "Tell me about Ran Pollak" |
| `get_team` | Get a manager's direct reports | "Who is on Tom's team?" |
| `search_people` | Search people by name or text | "Find someone named Adi" |
| `get_project` | Get details about a Jira project | "What is the CNV project?" |
| `get_category` | Get people in a work category | "Who works on AI/ML?" |
| `list_products` | List products by team size | "What products are being built?" |
| `list_categories` | List all work categories | "What categories exist?" |
| `get_site_overview` | High-level stats | "Give me an overview of the site" |

## API Endpoints

### `GET /api/modules/org-lens/datasets`

Returns loaded datasets with headcount per scope.

### `POST /api/modules/org-lens/chat`

Streams a chat response via SSE. Request body:

```json
{
  "message": "Who works on OpenShift?",
  "dataset": "manager_tgunders",
  "history": []
}
```

SSE events:
- `chunk` — partial text for progressive rendering
- `done` — stream complete

## Configuration

### Required

- **`GEMINI_API_KEY`** — Google Gemini API key. Set in `.env` locally, OpenShift secret in production. Never committed to code.

### Data Directory

Place dataset files in `data/org-lens/{scope_id}/`:

```
data/org-lens/
├── manager_tgunders/
│   ├── people_summaries_manager_tgunders.json
│   ├── people_categories_manager_tgunders.json
│   └── projects_manager_tgunders.json
└── manager_shuels/
    ├── people_summaries_manager_shuels.json
    └── ...
```

### Demo Mode

When `DEMO_MODE=true` and no real data is present, Org Lens loads fixture data (10 sample people) from `fixtures/`. If real data exists in `data/org-lens/`, it uses that even in demo mode. If no `GEMINI_API_KEY` is set, canned responses are returned instead.

## Architecture Context

Org Lens bridges two systems:

| | Org Pulse (this dashboard) | Site-Analysis (pipeline) |
|---|---|---|
| **Purpose** | Live operational dashboard | Batch discovery pipeline |
| **Data** | Raw metrics (Jira counts, story points) | AI-generated narratives (summaries, expertise) |
| **Updates** | Incremental refresh via API | Periodic full pipeline run |
| **AI usage** | Gemini for chat queries (this module) | Gemini for generating person summaries |

Org Pulse shows **live operational metrics**. Site-analysis discovers **what people actually do**. Org Lens bridges the two by making the pipeline's AI-generated insights queryable through chat.
